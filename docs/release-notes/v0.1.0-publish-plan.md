# v0.1.0 Publish Plan

This document describes the manual release plan for OpenVisi v0.1.0. It does not publish anything.

Do not publish if the finalization guard is misunderstood as final metrics. Do not claim production AI Visibility scoring.

## Manual Sequence

1. Clean runtime artifacts.
2. Run `npm run release:check`.
3. Run `npm run demo:mock`.
4. Clean demo artifacts with `npm run demo:mock:clean`.
5. Run `npm run release:rehearse`.
6. Clean rehearsal artifacts with `npm run release:rehearse:clean`.
7. Run `npm run release:rc-check`.
8. Review npm pack contents.
9. Review release notes.
10. Decide RC vs stable.
11. Only then manually tag, create a GitHub release, or publish if desired.

## Required Cautions

- No automatic `npm publish`.
- No automatic git tag creation.
- No automatic GitHub release creation.
- No real provider adapters are included.
- No final `metrics.json` is generated.
- No final AI Visibility Score is computed.
- No `scan-result.json` is generated.
- No final `report.md` or `report.html` is generated.

## Go Criteria

- `npm run release:check` passes.
- `npm run demo:mock` passes.
- `npm run release:rehearse` passes.
- `npm run release:rc-check` passes.
- Known limitations are reviewed.
- Tarball contents are reviewed.
- README and CHANGELOG are reviewed.

## Hold Criteria

- Any final scoring claim appears in docs.
- Any real provider claim appears in docs.
- Any runtime artifact directory remains in the repo root.
- Any generated `metrics.json`, `scan-result.json`, final `report.md`, or final `report.html` appears in release output.
