# OpenVisi v0.1.0 Release Notes

OpenVisi v0.1.0 is an early OSS release for machine-readable AI visibility diagnostics. It is focused on a local CLI workflow, transparent methodology, explainable analyzer output, and repeatable fixture foundations.

This release is not a production SaaS product, ranking prediction system, citation monitor, or provider-backed LLM evaluation tool.

## CLI MVP Capability

The CLI can scan a public target URL and generate local reports:

```bash
npm run cli -- scan https://example.com
```

Generated runtime reports are written to:

```text
reports/example-com/report.md
reports/example-com/report.json
reports/example-com/report.html
```

Curated demo report examples are committed under:

```text
examples/reports/example-com/
```

## Methodology Version

Current methodology version:

```text
0.1
```

Methodology version `0.1` represents an early heuristic diagnostic model based on crawlable public website signals. Scores are directional snapshots, not definitive ranking or citation models.

## Analyzer Output

The release includes analyzer output for:

- Entity Clarity
- Technical Discoverability
- Structured Data
- Content Chunkability
- Citation Readiness
- Prompt Simulation placeholder

Analyzer results include:

- score
- maturity label
- detected signals
- missing signals
- diagnostic interpretation
- suggested structural improvements
- evidence
- issues
- recommendations

## Fixture-Based Validation

The repository includes synthetic fixtures for directional analyzer validation:

- `fixtures/strong-entity-site/`
- `fixtures/weak-entity-site/`
- `fixtures/faq-heavy-site/`

Fixture tests verify directional behavior such as:

- strong entity fixture scoring above weak entity fixture
- FAQ-heavy fixture surfacing stronger content structure signals
- weak fixture exposing missing machine-readable and entity signals

The fixtures validate signal behavior. They do not represent real-world LLM ranking outcomes.

## Exploratory Benchmark Scaffolding

The release includes exploratory benchmark scaffolding:

- `benchmarks/exploratory/qms-vendors/`
- `benchmarks/exploratory/developer-tools/`

No benchmark data is included in this release. The folders document future structure, limitations, and interpretation cautions.

## Limitations

- The scoring model is heuristic and directional.
- Scans are snapshots and may change as websites change.
- The crawler is lightweight and may miss complex client-rendered content.
- Prompt simulation is a placeholder and does not run provider-backed checks.
- The methodology has not been validated against a large public benchmark dataset.
- Scores should not be interpreted as ranking, citation, or recommendation predictions.

## Non-Goals

OpenVisi v0.1.0 does not provide:

- ChatGPT, Claude, Gemini, or Perplexity ranking prediction
- citation guarantees
- provider API integrations
- hosted dashboard functionality
- production SaaS readiness
- automated content generation
- manipulation or growth-hacking workflows
