# OpenVisi v0.1.0 Release Notes

OpenVisi defines an open-source measurement layer for AI Visibility.

AI Visibility is the measurable presence, accuracy, citation quality, and competitive position of an entity across AI-generated answers.

## What v0.1.0 Includes

- Local CLI for running a mock artifact pipeline.
- Static crawler artifacts for AI-readable Structure and Machine-readable Trust evidence.
- Deterministic mock evaluator artifacts for answer signal plumbing.
- `measurement-inputs.json` composition.
- `metrics-draft.json` with transparent draft formulas.
- `metrics-review.json` review gate.
- `metrics-finalization.json` finalization guard.
- `debug-report.md` artifact pipeline summary.
- Golden artifact fixtures and contract regression tests.
- Local demo verifier:

```bash
npm run demo:mock
```

## What v0.1.0 Does Not Include

- No real provider adapters.
- No OpenAI, Anthropic, Google, Gemini, or other provider API calls.
- No API keys required.
- No `metrics.json`.
- No final AI Visibility Score.
- No `scan-result.json`.
- No production scoring.
- No SaaS dashboard or storage layer.

## Known Limitations

The mock evaluator exists for deterministic local validation. Mock evidence is not real LLM evidence.

Final metrics generation is expected to be blocked in v0.1.0 because the current pipeline does not include real provider evidence or narrative accuracy review.

See:

- [Known Limitations](v0.1.0-known-limitations.md)
- [RC Freeze Review](v0.1.0-rc-freeze-review.md)
- [Publish Plan](v0.1.0-publish-plan.md)

## Next Roadmap

- Continue artifact contract hardening.
- Add more fixture coverage.
- Design real provider adapters behind explicit environment-gated execution.
- Add human or real-LLM narrative accuracy review.
- Only then consider final metrics generation.
