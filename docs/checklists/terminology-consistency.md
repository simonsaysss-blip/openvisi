# Terminology Consistency Checklist

Purpose: Ensure all OpenVisi public artifacts use canonical vocabulary
consistently.

## Canonical Definition

- [ ] AI Visibility uses this exact definition: "AI Visibility is the measurable
      presence, accuracy, citation quality, and competitive position of an
      entity across AI-generated answers."
- [ ] The definition appears consistently in README, glossary, methodology, and
      report templates.

## Metric Names

- [ ] AI Visibility Score is not called by alternative names.
- [ ] Schema fields match canonical metric names.
- [ ] Report sections use canonical names.
- [ ] CLI output uses canonical JSON fields when reporting these metrics.

## Positioning

- [ ] "AI SEO" is not used as positioning.
- [ ] "SEO for ChatGPT" is not used as positioning.
- [ ] "rank higher in AI" is not used as positioning.
- [ ] "growth hack" is not used as positioning.
- [ ] "prompt optimization" is not used as positioning.
- [ ] "AI marketing automation" is not used as positioning.

## Benchmark Language

- [ ] Benchmark report title uses: The OpenVisi AI Visibility Benchmark
- [ ] Benchmark subtitle uses: Measuring Answer Presence, Entity Clarity,
      Citation Coverage, and Competitor Displacement across LLM-powered search
      surfaces.
