# Code of Conduct

OpenVisi uses a concise contributor standard: be respectful, be specific, and keep discussion focused on improving the project.

## Expected Behavior

- Use professional and welcoming language.
- Ask for evidence when claims affect methodology or artifact contracts.
- Keep feedback actionable and scoped to the project.
- Respect different levels of experience with TypeScript, CLI tooling, and AI evaluation methods.
- Represent OpenVisi accurately as AI Visibility infrastructure, not as a ranking manipulation tool.

## Unacceptable Behavior

- Harassment, insults, threats, or discriminatory language.
- Publishing private information without permission.
- Spam, off-topic promotion, or repeated unrelated requests.
- Misrepresenting mock outputs as real LLM evidence.
- Misrepresenting OpenVisi as a guaranteed ranking or citation tool.

## Enforcement

Maintainers may edit, hide, or remove comments, issues, pull requests, or other contributions that violate this code of conduct. Repeated or severe violations may lead to temporary or permanent bans from project spaces.

## Reporting

For sensitive conduct issues, use a private maintainer contact or GitHub security advisory flow if available. For non-sensitive concerns, open a concise issue with enough context for maintainers to act.
