# OpenVisi Field-test Article Series

This folder contains publishable drafts based on the OpenVisi field-test reports
in `benchmarks/field-tests/v0.1/`.

## Recommended Publishing Order

1. `developer-docs-ai-visibility-infrastructure.md`
2. `machine-readable-trust-regulated-saas.md`
3. `ai-companies-not-automatically-ai-readable.md`
4. `education-platforms-entity-clarity.md`
5. `marketplace-popularity-source-layer-clarity.md`
6. `cross-domain-ai-visibility-recap.md`

## Test Hypotheses

- Developer tools should respond to docs, structure, APIs, and AI-readable
  source language.
- Regulated SaaS should respond to Machine-readable Trust and evidence language.
- AI products should generate attention, but require careful methodology
  boundaries.
- Education should respond to Entity Clarity and course/category clarity.
- Commerce platforms should respond to marketplace/source-layer distinction.

## Publishing Guardrails

- Keep the phrase `crawl-diagnostic-v0.1` visible.
- Link to the relevant field-test report.
- Do not describe results as rankings.
- Do not claim live LLM answer behavior.
- Invite methodology feedback.
