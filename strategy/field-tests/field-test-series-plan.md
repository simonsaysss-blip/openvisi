# OpenVisi Field-test Content Series Plan

Goal: test which OpenVisi vocabulary and vertical angle earns the strongest
public response.

The series should publish small, transparent crawl-diagnostic snapshots across
different markets. Each post should link back to the benchmark notes in
`benchmarks/field-tests/v0.1/`.

## Publishing Sequence

1. Developer tools: docs as AI Visibility infrastructure.
2. Regulated SaaS: Machine-readable Trust in high-stakes categories.
3. AI products: AI-native companies are not automatically AI-readable.
4. Education: course and entity clarity for learning platforms.
5. Commerce platforms: marketplace popularity is not source-layer clarity.
6. Cross-domain recap: where OpenVisi language resonated.

## Metrics to Watch

- GitHub stars
- GitHub Discussion comments
- referral traffic to GitHub Pages
- LinkedIn reactions/comments
- X reposts/replies
- inbound questions about scanning a specific site
- language adoption: mentions of AI Visibility, Entity Clarity, AI-readable
  Structure, Machine-readable Trust, AI Citation Signals

## Core Guardrails

- Do not call the field tests rankings.
- Do not claim live LLM answer behavior.
- Keep `crawl-diagnostic-v0.1` visible.
- Explain `null` answer-level metrics clearly.
- Invite methodology feedback rather than positioning results as final.

## Best First Audience

Developer tools and regulated SaaS are the strongest first audiences because the
scan results produce clear, defensible narratives:

- Developer tools already understand docs, schemas, APIs, and source structure.
- Regulated SaaS buyers understand trust, evidence, compliance, and source
  authority.

AI products are high-attention but higher-risk. Publish that post after at least
one methodology-oriented post has established the measurement boundary.
