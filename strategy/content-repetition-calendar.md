# OpenVisi Content Repetition Calendar

This 8-week calendar repeats the same canonical vocabulary across docs, posts,
benchmarks, and community discussion.

## Week 1

Post idea: What is AI Visibility?

Core term: AI Visibility

Core message: AI Visibility is measurable through presence, accuracy, citation,
and competition.

Suggested channels: Blog, GitHub Discussions, LinkedIn, X.

Link target inside repo/docs: `docs/concepts/ai-visibility.md`

## Week 2

Post idea: Why AI gets your company wrong

Core term: Entity Clarity

Core message: AI systems misrepresent entities when category, audience,
positioning, and differentiation are unclear.

Suggested channels: Blog, GitHub Discussions, founder newsletter.

Link target inside repo/docs: `docs/concepts/entity-clarity.md`

## Week 3

Post idea: Why citations matter in AI answers

Core term: AI Citation Signals

Core message: Citation-ready public sources help AI-generated answers anchor
claims to evidence.

Suggested channels: Blog, GitHub Discussions, devrel newsletter, LinkedIn.

Link target inside repo/docs: `docs/concepts/ai-citation-signals.md`

## Week 4

Post idea: Your website is not AI-readable

Core term: AI-readable Structure

Core message: Clear docs, FAQs, stable URLs, schema metadata, and category pages
make source material easier to parse and cite.

Suggested channels: Blog, website docs, GitHub Discussions, X thread.

Link target inside repo/docs: `docs/concepts/ai-readable-structure.md`

## Week 5

Post idea: AI search is changing competitive discovery

Core term: Competitor Displacement

Core message: AI-generated answers can route attention toward competitors even
when the target brand has public source material.

Suggested channels: Blog, benchmark preview, LinkedIn, GitHub Discussions.

Link target inside repo/docs: `docs/concepts/competitor-displacement.md`

## Week 6

Post idea: How to measure AI answer presence

Core term: Answer Share

Core message: Presence must be interpreted with answer share because competitors

Suggested channels: Blog, methodology note, GitHub Discussions.

## Week 7

Post idea: Why trust must be machine-readable

Core term: Machine-readable Trust

Core message: Credibility, authority, freshness, and official source consistency
