# What should AI Visibility mean?

Discussion objective: pressure-test the canonical definition before OpenVisi
uses it across reports, schemas, benchmarks, and CLI output.

## Opening Post Draft

OpenVisi currently defines AI Visibility as:

> AI Visibility is the measurable presence, accuracy, citation quality, and
> competitive position of an entity across AI-generated answers.

The goal is to make this definition specific enough for open-source methodology,
but broad enough to cover brands, products, websites, open-source projects,
schools, documentation sites, and other entities.

The current OpenVisi model breaks AI Visibility into four dimensions:

- Presence: does an AI-generated answer mention the entity?
- Accuracy: does the answer describe the entity correctly?
- Citation: which sources does the answer use to understand or support the
  entity?
- Competition: does the answer route the user toward competitors or
  alternatives?

Questions for the community:

- Is the definition precise enough?
- Which word is most likely to be misunderstood?
- Is anything missing from the four dimensions?
- What would make this definition easier to measure reproducibly?
- What should OpenVisi avoid claiming while the methodology is still early?

## Desired Feedback

- Better wording for the canonical definition
- Examples where the definition works
- Examples where the definition fails
- Measurement edge cases
- Suggestions for benchmark methodology language
